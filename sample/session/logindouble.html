TLフレームワークのサンプルです．

Session の mode=double のテストです．
https，http の両方に同じCGIを設置し，テストしてください．

ボックスに整数を入力すると，そのデータを Session に記録します．
Session への記録は，https 側でのみ行えます．
値の取得は https，http の両方で行えます．

発行されるセッションIDは，http，https のそれぞれで異なるため， http のセッションIDが漏洩し使用されたとしても， https 領域の参照内容に影響を与えることは出来ません．
しかし，http 領域に関しては改ざんが可能なので， 重要な操作は https 領域でセッションを参照する必要があります．

ID:
現在のセッションデータは <&SESSIONDATA> です．
セッションIDは <&SESSIONID> です．
現在セッションデータは空です．
